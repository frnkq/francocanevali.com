First blog post. How I made the blog engine.

2020-04-17

I had a blog before, and it broke down without backups. I always searched for small blog engines, or micro PHP frameworks to handle HTTP requests and have a proper routing structure. The path I was following was always guided by the -don't know why- necessity of using an API design pattern. Well, I do know why. It's because we are in 2020 and it is an excellent standard to have around application development.

With all this being said, I was never confident with the solution I've found. Either I didn't like the theme of the engine, or didn't have the enthusiasm to change it, modifying front end components gets me bored really quick. It also always felt too much infrastructure to perform the intended task which is to upload to the internet, nothing more than plain text and a couple of images. A couple of times per month.
In addition, I was noticing that my work patterns were getting "professional" in a forced way -executivey, if you will-.

And from all that...this blog has raised. This blog engine, if it can be called that way, is file-based and a baby, as it was born yesterday. Briefly, the lifespan of a blog entry begins on an HTML template file -with the style making it's appearence in between style tags in the head of the HTML-, with 3 placeholders, one for the title of the post, the second one for the date of the post, and the third one for the post contents. This placeholders get replaced by a script ran to create a post with the intended data. The modified HTML file is placed in the "bin/" directory -spoiler: there isn't an "src/" directory-. And because everything is resided in a git directory, and because why not, a new commit is created with the title of the post entry, then pushed to the server, which triggers a post-receive hook that checks out the files into the production web directory. The wholesome index.php iterates the files of the "bin/" directory, displaying them as a list of links -that point to the physical html file- inside a div with a fixed height and an overflow: scroll; style property.
The glorious script that is ran to create the blog post is written in BASH, and performs the following operations in sequence:

->Pulls commits from the production server
->Stablishes the date of the post
->Creates a directory for the post
->Base html template is copied to post's directory
->An 'assets/' directory is created in the post's directory
->Post title is prompted
->Vim opens a temporary buffer
->Breaklines get replaced with HTML break lines
->Base html placeholders are replaced -title from prompt, date from system, contents from temporary buffer)
->Confirmation is asked in a do;while; as follows
While deploy = "N"
->Prompt the user to view the file -open the html with an internet browser-
->Prompt the user to edit the file
->Prompt the user to deploy
Done
->All files present in "assets/"'s directory of the post are copied to "bin/assets/"
->A new commit is created
->Commit is pushed


Ain't much, but it's honest work.



I plan to improve the engine a bit. Check if directories/files exists before operating with them -or hiding the error message lol-, creating the deletion functionality, amongst with edition, of the "compiled" post entries. Sort the code. Give it a name and feed it's first beer. What I like the most of this are two things: the first is that I'm having fun and not giving myself the opportunity to judge my code.
